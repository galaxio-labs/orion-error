One error language across layers, protocols, and teams.

orion-error keeps structure intact from first entry to boundary output: stable identities, explicit source handling, layered conversion, and protocol-aware rendering.

Stable business identities with #[derive(OrionError)]
First-entry source handling with source_err(...)
Reason-only layer mapping with conv_err()
Boundary export with report(), snapshot(), and exposure_snapshot()

Core paths

What to learn first

source_err(reason, detail)

Use when an error enters the structured system or when a new semantic boundary is created and the upstream error should remain the source.

conv_err()

Use when a lower-layer StructError only needs reason remapping, without introducing a new boundary.

to_err()

Use when a single reason enum value needs to become a structured error immediately.

with_field() / with_meta()

Use for chained context construction. Keep record_field() and record_meta() for existing mutable references.

let ctx = OperationContext::doing("load config")
    .with_field("path", "config.toml")
    .with_meta("component.name", "config_loader");
std::fs::read_to_string(path)
    .source_err(AppReason::system_error(), "read config failed")?
    .doing("read file")
    .with_context(&ctx);
