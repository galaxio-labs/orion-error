Tutorial

Start with stable identities, source boundaries, and context.

The shortest useful path is: define domain reasons, turn upstream failures into StructError<R>, add context, then expose a stable boundary view.

1. Define reasons

#[derive(Debug, Clone, PartialEq, From, OrionError)]
enum AppReason {
    #[orion_error(identity = "biz.invalid_request")]
    InvalidRequest,

    #[orion_error(transparent)]
    Uvs(UvsReason),
}

2. Build context fluently

let ctx = OperationContext::doing("load config")
    .with_field("path", "config.toml")
    .with_meta("component.name", "config_loader");

3. Enter the structured system

let content = std::fs::read_to_string(path)
    .source_err(AppReason::system_error(), "read config failed")?
    .doing("read file")
    .with_context(&ctx);

4. Remap without a new boundary

let user = repository.load_user(id)
    .conv_err()?;

5. Create a direct structured error

return Err(AppReason::validation_error()
    .to_err()
    .with_detail("missing order id")
    .with_field("field", "order_id"));
